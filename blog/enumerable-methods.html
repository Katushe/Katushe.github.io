Enumerable Method: Map

DBC Phase 0 Week 5

January 9, 2016

"Enumerable" is both a module, meaning its a group of methods packaged together, and a mixin, meaning it can be used in other classes such as arrays and hashes. It is comprised of enumerable methods, which are methods that allow for traversing through and searching objects.

One of these methods is map. It has both a non-destructive(plain old map) and destructive(map!) version. This means that map will create a copy of the array you call it on, so the original array is not affected by the method, while map! will modify the original array, so it won't exist in its original form anymore.

Map is used to transform an array. When you call the map method, it will be followed by a code block in which you specify the change you would like to make to the elements in the array. Map will then iterate through the array and apply that change to each element.

For example, say you have an array of integers and would like to multiply each integer by 2. Within the block, you declare a temporary variable for the current element, then specify that you want to multiply it by 2.

Now, nums will still have the original values, [2,5,9,15], while nums_times_two will have the transformed values of [4,10,18,30].

If you use the destructive version, map! then both nums and nums_times_two will have the transformed values of [4,10,18,30].
